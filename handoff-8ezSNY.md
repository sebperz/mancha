# Handoff: Mancha — Implementation

## What is Mancha

A cross-platform desktop app (Linux/Mac/Windows) for scientists to classify and measure cells in microscopy images. Masks are pre-generated by Cellpose-SAM (`_masks.tif` files). The scientist clicks masks to assign Cell Types, then exports a CSV with morphological measurements.

## Artifacts already created

- **`CONTEXT.md`** — domain glossary (11 terms: Source Image, Mask, Cell Type, Classification, Calibration, etc.)
- **`docs/adr/0001-session-state-in-data-folder.md`** — why `.mancha` config lives in the image folder, not `~/.config`
- **`AGENTS.md`** — agent skills config (GitHub Issues, triage labels, domain docs)

## Tech stack

- **UI**: PySide6 (Qt)
- **Image processing**: skimage (`regionprops` for measurements)
- **Mask format**: `tifffile` for reading `_masks.tif` (2D uint16, background=0, contiguous IDs from 1)
- **Packaging**: PyInstaller (single executable per platform)
- **Config**: JSON `.mancha` file in the image folder

## Key design decisions (not in CONTEXT.md)

| # | Topic | Decision |
|---|-------|----------|
| Layout | Sidebar (Cell Types panel, image list, progress bar, nav buttons) + main viewport (QGraphicsView with smooth zoom/pan) + status bar + menu bar | |
| Calibration | "N pixels = M units" entered once per folder. Linear: `factor = units/px`, area: `factor²`. Changeable later via menu. |
| Mask visualization | Semi-transparent fills (30% opacity) + solid darker outlines per Cell Type color. Unclassified = light gray. |
| Mask states | Unclassified ↔ Cell Type N. Left-click with type selected to classify, right-click to unclassify. No "Unknown" type. |
| Cell Type colors | Auto-assigned from a fixed distinguishable palette (Tableau 10). User cannot change colors. |
| Cell Type CRUD | Add, rename, delete. Renaming preserves assignments. Deleting shows confirm dialog ("assigned to N masks — unclassify them?") then reverts them. Zero types allowed. |
| Navigation | One image pair at a time. Image list sidebar with click-to-jump. ← → Space for prev/next. Number keys 1-9 to select Cell Type. |
| Mask click mapping | `mask_id = masks[y, x]` — direct lookup from the mask array. |
| Mask inspection | Click a mask to select it — sidebar panel shows its measurements in real time. |
| Pair matching | Masks are the anchor: find `_masks.tif` files, strip `_masks`, match to `.jpg/.png/.tif` source. |
| Size mismatch | Skip the pair, log warning. |
| Zero-mask images | Show normally with "0 masks detected" — scientist can still view, verify, navigate past. |
| Source formats | `.jpg`, `.png`, `.tif` for source images. Masks always `.tif`. |
| Edge cells | Measured, flagged with `edge_cell` boolean in CSV. |
| Small masks | All masks kept (no filtering). |
| CSV columns | `image`, `mask_id`, `cell_type`, `area`, `major_axis`, `minor_axis`, `feret_max`, `feret_min`, `perimeter`, `circularity` (4πA/P²), `calibration`, `edge_cell` |
| CSV naming | `mancha-YYYYMMDD.csv` timestamped. Export via File menu or Ctrl+E. Only classified masks. |
| Undo/Redo | Full Ctrl+Z / Ctrl+Y stack (per-session, not persisted). |
| Session resume | Auto-resume silently when re-opening a folder (config already in `.mancha`). |
| Folder changes | Manual "Reload Folder" button — no file watcher. |
| Window state | Saved per-folder in `.mancha` (size, position, sidebar width, zoom). |
| HiDPI | Qt auto-scaling. |
| Empty state | Welcome screen with "Select Image Folder" button + one-line description. |
| Performance | Lazy load current image pair + prefetch adjacent ones in background thread. |
| Test data | `sampleData/` directory (18 image pairs) in the repo. |
| Circularity | `4π × area / perimeter²` (1.0 = perfect circle). |

## Implementation plan (TDD vertical slices)

Build in this order, one slice at a time, red-green-refactor:

1. **Folder loading** — Scan folder for `_masks.tif`, derive source images, validate dimensions, report mismatches/missing pairs.
2. **Image display + mask overlay** — QGraphicsView rendering source image with semi-transparent mask fills and outlines.
3. **Click-to-classify** — Left/right clicks on masks, mask ID lookup, state machine (Unclassified ↔ Cell Type).
4. **Cell Type CRUD** — Sidebar panel: list types with color swatches, add/rename/delete with confirmation.
5. **Calibration + measurements** — skimage regionprops, unit conversion, per-mask measurement computation.
6. **Config persistence** — `.mancha` JSON auto-save (Cell Types, calibration, classifications, window state).
7. **CSV export** — Generate timestamped CSV with all classified masks' measurements.
8. **Session resume** — Load `.mancha` on folder open, restore all state.
9. **Undo/redo** — Command stack for classification actions.
10. **Navigation + progress** — Image list sidebar, prev/next buttons, keyboard shortcuts, progress tracking.
