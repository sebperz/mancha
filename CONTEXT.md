# Mancha

A desktop tool for scientists to classify and measure cells in microscopy images, using masks pre-generated by Cellpose-SAM.

## Language

### Core concepts

**Source Image**:
A microscopy image file (`.jpg`, `.png`, or `.tif`) depicting cells.
_Avoid_: original image, raw image

**Mask Image**:
A `_masks.tif` file generated by Cellpose-SAM, containing integer-labeled segmented regions matching a Source Image.
_Avoid_: segmentation file, label image

**Image Pair**:
A Source Image and its corresponding Mask Image, matched by filename stem. Each Pair shares the same pixel dimensions.
_Avoid_: image set, image group

**Mask**:
A contiguous segmented region within a Mask Image, identified by a positive integer Mask ID. Represents a single detected cell.
_Avoid_: ROI, region, segment, blob

**Mask ID**:
The positive integer label (1, 2, 3, …) uniquely identifying a Mask within its Mask Image.
_Avoid_: label, index, region ID

### Classification

**Cell Type**:
A user-defined category that the scientist assigns to Masks. Each Cell Type has a name and an auto-assigned display color.
_Avoid_: class, label, category

**Classification**:
The act of assigning a Cell Type to a specific Mask.
_Avoid_: labeling, tagging, annotating

**Unclassified**:
A Mask that has not been assigned a Cell Type.
_Avoid_: unlabeled, unassigned, pending

### Measurement

**Calibration**:
The conversion factor between image pixels and real-world measurement units, expressed as "N pixels = M units" (e.g., "100 pixels = 10 µm").
_Avoid_: scale, pixel ratio, magnification

**Measurement**:
A set of computed morphological metrics for a Mask (area, major axis, minor axis, Feret diameters, perimeter, circularity), expressed in calibrated real-world units.
_Avoid_: statistics, metrics, features

**Edge Cell**:
A Mask that touches the border of the Source Image and whose measurements may therefore be incomplete.
_Avoid_: border cell, boundary cell

### Session

**Session**:
A working session on a folder containing Image Pairs. State (Cell Types, Calibration, Classifications, window layout) is persisted automatically to a `.mancha` config file in that folder.
_Avoid_: project, workspace

**CSV Export**:
A timestamped `.csv` file written to the folder, containing one row per classified Mask with its image filename, Mask ID, Cell Type, and Measurements.
_Avoid_: results file, data export

## Relationships

- A **folder** contains one or more **Image Pairs**.
- An **Image Pair** contains one or more **Masks**.
- A **Mask** has zero or one **Classification**.
- A **Cell Type** can be assigned to zero or more **Masks**.
- A **folder** has one **Calibration**.
- A **Session** produces zero or more **CSV Exports**.

## Example dialogue

> **Dev:** "If a scientist deletes a **Cell Type** that's already assigned to 12 **Masks**, what happens to those **Masks**?"
> **Domain expert:** "They become **Unclassified**. The app shows a confirmation dialog first — 'Cell Type 1 is assigned to 12 masks. Delete it and unclassify those masks?'"
>
> **Dev:** "And if a **Mask** touches the image border, do we still **Measure** it?"
> **Domain expert:** "Yes, but we flag it as an **Edge Cell** in the **CSV Export**. The scientist can decide whether to include or exclude those rows downstream."

## Flagged ambiguities

- "Unknown" was initially proposed as a Cell Type distinct from Unclassified, meaning "the scientist looked at this cell and couldn't identify it." Resolved: removed from scope to reduce complexity. All non-classified masks are Unclassified.
